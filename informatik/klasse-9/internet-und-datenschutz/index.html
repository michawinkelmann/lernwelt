Internet und Datenschutz

Du tippst eine Nachricht, drückst auf „Senden“ — und einen Wimpernschlag später liegt sie auf einem Handy am anderen Ende der Welt. Was passiert dazwischen? Auf dieser Seite verfolgst du den Weg deiner Daten: vom Heimnetz über Router und DNS bis zum Server. Und weil dabei unterwegs und am Ziel Datenspuren entstehen, geht es im zweiten Teil darum, was andere über dich erfahren können — und wie du deine Daten schützt.

1. Wie das Internet aufgebaut ist

Wenn du eine Webseite öffnest, arbeiten zwei Computer zusammen. Dein Gerät ist der Client (englisch für „Kunde“): Es fragt etwas an. Am anderen Ende antwortet ein Server („Diener“) — ein Computer, der rund um die Uhr läuft und Webseiten, Videos oder Nachrichten bereithält. Dazwischen liegt eine Kette von Routern: Geräte, die Daten Stück für Stück in Richtung Ziel weiterreichen, wie Poststellen, die ein Paket immer eine Station weitergeben. Die erste dieser Stationen kennst du gut: deinen WLAN-Router zu Hause.

Das Internet selbst ist kein einzelnes Netz mit einem Hauptcomputer in der Mitte, sondern ein Zusammenschluss vieler Netze: Schul-, Firmen- und Anbieternetze auf der ganzen Welt, an unzähligen Stellen miteinander verknüpft. Daher der Name: Internet, das „Netz der Netze“. Diese Bauweise macht es robust — fällt eine Verbindung aus, finden die Daten meist einen anderen Weg.

Merke

Client: fragt Dienste an (z. B. dein Browser).
Server: stellt Dienste bereit (Webseiten, Videos, Nachrichten).
Router: leitet Datenpakete in Richtung Ziel weiter.
Internet: weltweiter Verbund vieler Netze — ohne Zentrale und dadurch ausfallsicher.

2. Adressen im Netz: IP-Adressen und DNS

Damit Daten ankommen, braucht jedes Gerät im Internet eine eindeutige Adresse: die IP-Adresse (IP = Internet Protocol). Eine IPv4-Adresse besteht aus vier Zahlen von 0 bis 255, durch Punkte getrennt — zum Beispiel 192.168.2.107. Die Obergrenze 255 kennst du von der Seite Daten und Codierung: Jede der vier Zahlen ist genau ein Byte. Insgesamt sind das rund $4{,}3$ Milliarden mögliche Adressen — klingt viel, ist aber längst knapp geworden. Deshalb gibt es den Nachfolger IPv6 mit praktisch unerschöpflich vielen Adressen.

Zahlenkolonnen merkt sich allerdings niemand gern. Deshalb gibt es das DNS (Domain Name System) — das „Telefonbuch des Internets“: Es übersetzt Namen wie www.beispielschule.de in die zugehörige IP-Adresse. Du tippst den Namen, dein Browser schlägt die Nummer nach.

Beispiel: Was beim Aufruf einer Webseite passiert

1. Du tippst www.beispielschule.de in die Adresszeile.
2. Dein Browser fragt einen DNS-Server: „Welche IP-Adresse gehört zu diesem Namen?“
3. Der DNS-Server antwortet, etwa mit 203.0.113.42.
4. Erst jetzt schickt dein Browser die eigentliche Anfrage an diese Adresse — und der Webserver antwortet mit der Seite.

Verständnis-Check: Wäre das Internet ohne DNS unbenutzbar?

Technisch würde es weiter funktionieren — entscheidend ist am Ende nur die IP-Adresse. Aber du müsstest dir für jede Website die Zahlenfolge merken und eintippen, etwa 203.0.113.42 statt eines Namens. Das DNS ist also vor allem eine riesige Komfortfunktion: Namen für Menschen, Nummern für Maschinen.

3. Datenpakete und Protokolle

Eine Datei reist nicht am Stück durchs Netz. Sie wird in viele kleine Datenpakete zerlegt — typisch sind etwa 1500 Byte pro Paket. Jedes Paket bekommt einen „Adressaufkleber“ mit Absender- und Ziel-IP sowie einer laufenden Nummer und wird einzeln auf die Reise geschickt. Die Router unterwegs entscheiden für jedes Paket neu, welche Verbindung gerade frei ist — Pakete derselben Datei können also verschiedene Wege nehmen und in bunter Reihenfolge ankommen. Der Empfänger sortiert sie anhand der Nummern und setzt die Datei wieder zusammen, wie ein Puzzle aus nummerierten Teilen.

Damit Milliarden Geräte sich dabei verstehen, halten sich alle an Protokolle — fest vereinbarte Regeln für den Datenaustausch. Die zwei wichtigsten bilden ein Team: IP kümmert sich um Adressen und Wege der einzelnen Pakete. TCP nummeriert die Pakete, prüft beim Empfänger auf Vollständigkeit und fordert verloren gegangene Pakete einfach neu an. Erst dieses Doppel „TCP/IP“ macht aus unzuverlässigen Einzelpaketen eine zuverlässige Übertragung.

Merke

Ein Protokoll ist eine Sammlung fester Regeln für den Datenaustausch. Im Internet arbeiten Protokolle zusammen: IP adressiert und leitet Pakete, TCP sichert Vollständigkeit und Reihenfolge, und HTTP/HTTPS ist die Sprache zwischen Browser und Webserver.

Ganz oben sitzt nämlich HTTP (Hypertext Transfer Protocol): die Sprache, in der Browser Seiten anfordern und Server sie liefern. HTTP allein hat einen Haken — alles reist im Klartext, jede Zwischenstation könnte mitlesen. Deshalb ist heute HTTPS Standard: Das S steht für secure, die Inhalte werden verschlüsselt. Router unterwegs sehen dann nur noch Zeichensalat, und das Schloss-Symbol in der Adresszeile zeigt dir die verschlüsselte Verbindung an. Wie diese Verschlüsselung im Kern funktioniert, skizziert die Vertiefung am Ende der Seite.

Beispiel: Wie lange dauert ein Download?

Datenraten misst man in Mbit/s (Megabit pro Sekunde), Dateigrößen meist in MB (Megabyte) — und 1 Byte = 8 Bit. Ein Video von 30 MB über eine 60-Mbit/s-Leitung: erst umrechnen, $30 \cdot 8 = 240$ Mbit, dann durch die Datenrate teilen: $240 : 60 = 4$ — der Download dauert rund 4 Sekunden.

Verständnis-Check: Du lädst ein 10-MB-Foto über eine 40-Mbit/s-Leitung. Wie lange dauert das mindestens?

Umrechnen: $10 \cdot 8 = 80$ Mbit. Durch die Datenrate teilen: $80 : 40 = 2$ — mindestens 2 Sekunden. In Wirklichkeit etwas mehr, weil die volle Datenrate selten erreicht wird.

4. Datenspuren im Alltag

Damit das alles funktioniert, müssen Geräte ständig Adressen und Verbindungsdaten austauschen — und genau dadurch entstehen Datenspuren. Das ist zunächst völlig normal und kein Grund zur Panik. Wichtig ist nur: Wer weiß, welche Spuren entstehen, kann bewusst entscheiden, was er preisgibt.

Server-Logs: Jeder Webserver sieht zwingend deine IP-Adresse — sonst könnte er dir die Antwort gar nicht zurückschicken. Üblicherweise notiert er in seinem Logbuch außerdem Datum und Uhrzeit, die aufgerufene Seite und welcher Browser auf welchem Gerätetyp anfragt.

Cookies und Tracking: Cookies sind kleine Textdaten, die eine Website in deinem Browser ablegt und beim nächsten Besuch zurückbekommt. Das ist oft praktisch — so bleibt der Warenkorb im Online-Shop gefüllt und du musst dich nicht ständig neu anmelden. Drittanbieter-Cookies stammen dagegen nicht von der besuchten Seite selbst, sondern zum Beispiel von einem Werbenetzwerk, das auf vielen Websites eingebunden ist. Es erkennt dich überall am selben Cookie wieder — so entsteht Tracking: ein Interessenprofil über viele Seiten hinweg, mit dem Werbung und Empfehlungen personalisiert werden. Deshalb „verfolgen“ dich die einmal angeschauten Sneaker tagelang als Werbung.

Metadaten: Selbst wenn Inhalte verschlüsselt sind, bleibt sichtbar, dass kommuniziert wird: wer mit wem, wann, wie oft, wie lange. Das klingt harmlos, verrät aber einiges über Beziehungen und Gewohnheiten — etwa wer dir nahesteht oder wann du meistens online bist.

Beispiel: Ein Nachmittag im Netz — und seine Spuren

Messenger: Deine Nachrichten sind Ende-zu-Ende-verschlüsselt — aber der Anbieter kann speichern, wann und wie oft du mit wem schreibst (Metadaten).
Online-Shop: Der Warenkorb funktioniert per Cookie. Das eingebundene Werbenetzwerk merkt sich die angesehenen Sneaker — und zeigt sie dir abends auf ganz anderen Seiten wieder.
Social Media: Jedes Like und jede Verweildauer fließt in dein Interessenprofil ein — daraus entsteht dein personalisierter Feed.

Verständnis-Check: Deine Messenger-Nachrichten sind Ende-zu-Ende-verschlüsselt. Welche Daten können trotzdem anfallen?

Die Metadaten: wer mit wem schreibt, wann, wie oft und wie lange — dazu technische Angaben wie IP-Adresse und Gerät. Den Inhalt kann der Anbieter nicht lesen, aber schon aus den Verbindungsdaten lässt sich ablesen, wer dir nahesteht und wann du aktiv bist.

5. Datenschutz und Selbstschutz

Beim Datenschutz geht es um personenbezogene Daten: alle Angaben, die sich einer bestimmten Person zuordnen lassen — Name, Adresse, Geburtsdatum, Fotos, Standortverlauf, häufig auch die IP-Adresse. Über solche Daten sollst du selbst bestimmen können.

Deine Rechte nach der DSGVO (vereinfacht)

In Europa schützt dich die Datenschutz-Grundverordnung (DSGVO). Drei Rechte solltest du kennen:
Einwilligung: Personenbezogene Daten dürfen in der Regel nur verarbeitet werden, wenn du zustimmst (bei Jüngeren zusätzlich die Eltern) — und du darfst deine Zustimmung widerrufen.
Auskunft: Du darfst von jedem Anbieter erfahren, welche Daten er über dich gespeichert hat.
Löschung: Du kannst verlangen, dass deine Daten gelöscht werden („Recht auf Vergessenwerden“).

Neben diesen Rechten gibt es vieles, das du selbst in der Hand hast:

Sichere Passwörter: Länge schlägt Kompliziertheit — ein Fantasiesatz wie „BlauerToasterSingt7Lieder!“ ist stark und merkbar. Wichtig: für jeden wichtigen Dienst ein eigenes Passwort, damit ein geknacktes Konto nicht alle anderen mitreißt. Ein Passwort-Manager hilft beim Merken.

Zwei-Faktor-Authentisierung (2FA): Beim Anmelden brauchst du zusätzlich zum Passwort einen zweiten Nachweis — etwa einen Code auf dem Handy. Selbst ein gestohlenes Passwort reicht Angreifern dann nicht mehr.

Datensparsamkeit: Gib nur an, was für den Zweck wirklich nötig ist. Pflichtfelder ausfüllen, freiwillige Felder hinterfragen, App-Berechtigungen prüfen — braucht ein Quiz wirklich Zugriff auf dein Mikrofon? Und vor dem Posten kurz überlegen: Das Netz vergisst nur schwer.

Du musst nicht alles auf einmal umsetzen. Schon ein langes, eigenes Passwort für die wichtigsten Konten plus 2FA hebt deine Sicherheit deutlich — der Rest kommt Stück für Stück.

Vertiefung: Wie HTTPS verschlüsselt — der Trick mit dem offenen Vorhängeschloss

Verschlüsseln können Browser und Server nur mit einem gemeinsamen Geheimnis — einem Schlüssel. Aber wie vereinbart man ein Geheimnis über eine Leitung, auf der jeder mitlesen könnte? Die Lösung heißt asymmetrische Verschlüsselung: Der Server besitzt ein Schlüsselpaar. Den öffentlichen Schlüssel darf jeder kennen — er wirkt wie ein offenes Vorhängeschloss: Jeder kann damit eine Kiste verschließen, also eine Nachricht verschlüsseln. Öffnen kann sie nur der private Schlüssel, und der verlässt den Server nie.

Beim Aufbau einer HTTPS-Verbindung schickt der Server zuerst sein „offenes Schloss“ (genauer: ein Zertifikat mit dem öffentlichen Schlüssel, das zugleich die Identität des Servers bestätigt). Dein Browser würfelt einen frischen Sitzungsschlüssel, verschließt ihn mit dem Schloss und schickt ihn zurück — Mitlesen zwecklos. Ab jetzt verschlüsseln beide Seiten mit dem gemeinsamen Sitzungsschlüssel symmetrisch weiter, weil das viel schneller ist. Beachte die Grenze des Schloss-Symbols: Es bestätigt die verschlüsselte Verbindung zu einem geprüften Server — nicht, dass dessen Inhalte seriös sind.
