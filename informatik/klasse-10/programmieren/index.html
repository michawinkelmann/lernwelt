Programmieren

Eine Idee als eindeutige Schrittfolge aufschreiben — das kennst du von den Algorithmen. Jetzt geht es einen Schritt weiter: Du übersetzt solche Schrittfolgen in eine Sprache, die der Computer ausführen kann. Auf dieser Seite liest du erste Python-Programme: Variablen speichern Werte, Verzweigungen treffen Entscheidungen, Schleifen übernehmen das Wiederholen, und Funktionen halten alles zusammen. Du brauchst dafür keinen Computer — Lesen, Nachvollziehen und Vorhersagen genügt.

1. Vom Algorithmus zum Programm

Ein Algorithmus ist eine eindeutige Schrittfolge, die ein Problem löst — aufgeschrieben als Text, Stichpunktliste oder Ablaufdiagramm (mehr dazu auf der Seite Algorithmen-Grundlagen). Solange ein Mensch die Schritte ausführt, darf die Beschreibung locker bleiben, denn Menschen denken mit. Ein Computer denkt nicht mit: Er braucht jede Anweisung exakt und eindeutig.

Beim Planen auf Papier hilft Pseudocode — eine halb-formale Mischung aus kurzen Anweisungen und Alltagssprache, ohne feste Regeln. Ein Programm dagegen ist ein Algorithmus in einer Programmiersprache: Sie hat eine strenge Grammatik (Syntax), damit der Computer den Text Schritt für Schritt ausführen kann. Wir verwenden hier Python, eine weit verbreitete und besonders gut lesbare Sprache.

Vom Algorithmus zum Programm

Pseudocode: Skizze des Algorithmus für Menschen — formlos, nicht ausführbar.
Programmiersprache: Sprache mit strenger Grammatik, die der Computer versteht (z. B. Python).
Programm: der Algorithmus, fertig in einer Programmiersprache aufgeschrieben — jetzt ausführbar.

Beispiel: Dreimal grüßen — als Pseudocode und als Python-Programm

Pseudocode:
wiederhole 3-mal:
  gib „Hallo!“ aus

for i in range(3):
    print("Hallo!")

Ausgabe: dreimal untereinander Hallo! — beide Fassungen beschreiben denselben Algorithmus, aber nur die Python-Fassung kann der Computer ausführen.

Die gute Nachricht: Um Programme zu lesen und ihr Verhalten vorherzusagen, musst du Python nicht fehlerfrei schreiben können — genau dieses Lesen übst du auf dieser Seite. Das wichtigste Werkzeug dabei ist der Schreibtischtest: Du spielst das Programm Zeile für Zeile von Hand durch und notierst, was sich ändert.

2. Variablen und Datentypen

Programme arbeiten mit Daten — und die wohnen in Variablen. Eine Variable ist ein beschrifteter Speicherplatz: Sie hat einen Namen und einen Wert. Den Wert legt eine Zuweisung mit dem Gleichheitszeichen fest:

name = "Mia"
alter = 16
groesse = 1.72
musik_an = True

Die vier Zeilen zeigen zugleich die wichtigsten Datentypen:

  • Text (String, str): Zeichen in Anführungszeichen, z. B. "Mia".
  • Ganze Zahl (int): z. B. 16 — zum Zählen und Rechnen.
  • Kommazahl (float): z. B. 1.72 — Achtung: Python schreibt den Dezimalpunkt.
  • Wahrheitswert (bool): nur True oder False — perfekt für Ja/Nein-Fragen.
Das Gleichheitszeichen ist ein Befehl

alter = 16 ist keine mathematische Aussage, sondern eine Zuweisung: „Speichere den Wert 16 unter dem Namen alter.“ Es wird immer zuerst die rechte Seite ausgerechnet und das Ergebnis dann links gespeichert. Deshalb ist auch alter = alter + 1 erlaubt — es erhöht alter um 1.

Beispiel: Eine Variable ändert ihren Wert 
alter = 16
alter = alter + 1
print(alter)

Ausgabe: 17. Der Befehl print(...) zeigt einen Wert auf dem Bildschirm an.

Wähle aussagekräftige Namen: anzahl_schueler sagt sofort, worum es geht — x7 nicht. Üblich sind Kleinbuchstaben mit Unterstrichen zwischen den Wörtern; Umlaute und Leerzeichen sind in Namen tabu. In großen Programmen entscheiden gute Namen darüber, ob der Code lesbar oder rätselhaft ist.

Verständnis-Check: Worin unterscheiden sich 42 und "42"?

42 ist eine ganze Zahl (int) — mit ihr kann man rechnen: 42 + 1 ergibt 43. "42" ist wegen der Anführungszeichen ein Text (String). Texte werden mit + aneinandergehängt: "42" + "1" ergibt den Text "421", nicht 43!

3. Verzweigungen: if, elif, else

Bisher läuft jedes Programm stur von oben nach unten. Mit einer Verzweigung trifft es Entscheidungen: if prüft eine Bedingung — ist sie wahr, wird der eingerückte Block darunter ausgeführt, sonst der else-Block.

punkte = 64
if punkte >= 50:
    print("bestanden")
else:
    print("nicht bestanden")

Hier ist 64 >= 50 wahr — die Ausgabe lautet bestanden. Bedingungen baust du mit Vergleichsoperatoren: == (gleich), != (ungleich), <, <=, >, >=. Achtung, beliebte Falle: == vergleicht — das einfache = weist zu!

Einrückung ist Pflicht

Hinter if, elif und else steht ein Doppelpunkt, und alle Zeilen, die zum Zweig gehören, werden um 4 Leerzeichen eingerückt. In Python ist die Einrückung Teil der Sprache: Sie legt fest, welche Zeilen zusammengehören. Falsche Einrückung ändert die Bedeutung des Programms — oder stoppt es mit einer Fehlermeldung.

Mit elif (kurz für „else if“) prüfst du mehrere Bedingungen nacheinander — zum Beispiel Notenstufen:

punkte = 92
if punkte >= 90:
    stufe = "sehr gut"
elif punkte >= 75:
    stufe = "gut"
elif punkte >= 50:
    stufe = "bestanden"
else:
    stufe = "nicht bestanden"
print(stufe)

Python prüft von oben nach unten und führt den ersten Zweig aus, dessen Bedingung wahr ist — alle weiteren werden übersprungen. Mit punkte = 92 gewinnt schon die erste Bedingung: Ausgabe sehr gut. Und genau ein Zweig läuft immer, notfalls der else-Zweig.

Verständnis-Check: Was gibt das Notenprogramm für punkte = 75 aus?

gut. Die erste Bedingung 75 >= 90 ist falsch, die zweite 75 >= 75 ist wahr — also wird stufe = "gut" ausgeführt. Dass auch 75 >= 50 wahr wäre, spielt keine Rolle mehr: Nach dem ersten Treffer überspringt Python alle weiteren Zweige.

4. Schleifen: for und while

Computer sind unschlagbar im Wiederholen. Die Zählschleife for durchläuft eine Zahlenreihe: range(1, 5) liefert die Zahlen 1, 2, 3, 4 — der Endwert ist nicht mehr dabei.

for zahl in range(1, 5):
    print(zahl)

Ausgabe: die Zahlen 1 bis 4, jede in einer eigenen Zeile. Die Schleifenvariable zahl nimmt in jedem Durchlauf den nächsten Wert an, und der eingerückte Schleifenkörper wird jedes Mal ausgeführt.

Beispiel mit Schreibtischtest: eine Summe aufaddieren 
summe = 0
for i in range(1, 5):
    summe = summe + i
print(summe)
Schreibtischtest: die Variablenwerte am Ende jedes Durchlaufs — Ausgabe: 10.
Durchlaufisumme
111
223
336
4410

Der Schreibtischtest — Zeile für Zeile ausführen, Werte notieren — ist das wichtigste Werkzeug, um Schleifen zu verstehen. Am Ende gibt print(summe) den Wert 10 aus.

Die bedingte Schleife while wiederholt, solange eine Bedingung wahr ist — praktisch, wenn die Anzahl der Durchläufe vorher nicht feststeht. Ein Klassiker: Eine Seerose verdoppelt jeden Tag ihre Fläche. Nach wie vielen Tagen übersteigt sie 100 Quadratzentimeter?

flaeche = 1
tage = 0
while flaeche < 100:
    flaeche = flaeche * 2
    tage = tage + 1
print(tage)

Die Fläche wächst 1 → 2 → 4 → 8 → 16 → 32 → 64 → 128; erst bei 128 ist die Bedingung flaeche < 100 falsch, und die Schleife endet. Ausgabe: 7. Vorsicht: Wird die Bedingung niemals falsch, läuft die Schleife endlos — einer der beliebtesten Programmierfehler überhaupt.

Faustregel: for, wenn die Anzahl der Wiederholungen feststeht — while, wenn eine Bedingung über das Ende entscheidet.

Verständnis-Check: Grenze 10 statt 100 — welche Zahl wird ausgegeben?

4. Schreibtischtest für while flaeche < 10: — die Fläche wächst 1 → 2 (Tag 1) → 4 (Tag 2) → 8 (Tag 3) → 16 (Tag 4). Bei 8 ist 8 < 10 noch wahr, also wird ein viertes Mal verdoppelt; bei 16 ist die Bedingung falsch und die Schleife endet. Ausgabe: 4.

5. Funktionen: Programme strukturieren

Wenn Programme wachsen, braucht es Ordnung. Eine Funktion bündelt einen Arbeitsschritt unter einem eigenen Namen: Mit def wird sie definiert, über ihre Parameter bekommt sie Eingaben, und mit return liefert sie ihr Ergebnis zurück — den Rückgabewert. Definiert wird einmal, aufgerufen beliebig oft:

def begruessung(name):
    return "Hallo, " + name + "!"

print(begruessung("Mia"))
print(begruessung("Ben"))

Ausgabe: Hallo, Mia! und Hallo, Ben!. Beim Aufruf begruessung("Mia") erhält der Parameter name den Wert "Mia"; das mit return zurückgegebene Ergebnis wandert an die Stelle des Aufrufs und wird dort ausgegeben.

Richtig stark werden Funktionen im Zusammenspiel: Ein Programm entsteht aus kleinen, verständlichen Bausteinen, die einander aufrufen.

def quadrat(x):
    return x * x

def summe_quadrate(n):
    summe = 0
    for i in range(1, n + 1):
        summe = summe + quadrat(i)
    return summe

print(summe_quadrate(3))

Hier nutzt summe_quadrate die Funktion quadrat und eine for-Schleife: 1 + 4 + 9 — Ausgabe 14. Variablen, Verzweigungen, Schleifen, Funktionen: Aus genau diesen Bausteinen bestehen auch riesige Programme.

Warum Funktionen?

Wiederverwenden statt kopieren: einmal schreiben, beliebig oft aufrufen.
Lesbarkeit: Gute Funktionsnamen machen ein Programm fast wie einen Text lesbar.
Testbarkeit: Jede Funktion lässt sich einzeln prüfen — mehr dazu in der Vertiefung. Auch die Verfahren von der Seite Suchen und Sortieren verpackt man in der Praxis genau so in Funktionen.

Vertiefung: Fehler finden und testen

Fehler gehören zum Programmieren dazu — Profis unterscheiden zwei Sorten. Ein Syntaxfehler verletzt die Grammatik der Sprache: Bei print("Hallo" fehlt die schließende Klammer, und Python bricht sofort mit einer Fehlermeldung ab, noch bevor irgendetwas läuft. Solche Fehler sind lästig, aber gutmütig: Die Meldung nennt meist die betroffene Zeile, und die Korrektur ist schnell gefunden.

Heimtückischer ist der logische Fehler: Das Programm läuft ohne Fehlermeldung durch — rechnet aber das Falsche. Ein Beispiel: Der Mittelwert von 4 und 7 soll 5.5 ergeben.

a = 4
b = 7
mittel = a + b / 2
print(mittel)

Ausgabe: 7.5 statt 5.5! Python hält sich an Punkt-vor-Strich und rechnet a + (b / 2) = 4 + 3.5. Keine Fehlermeldung warnt davor — nur das Ergebnis ist falsch. Die Korrektur: mittel = (a + b) / 2 liefert 5.5.

Wie findet man so etwas? Durch systematisches Testen: Du überlegst dir vorab Testfälle mit bekanntem Ergebnis („der Mittelwert von 4 und 7 muss 5.5 sein“) und prüfst das Programm dagegen — wie ein Schreibtischtest mit klarer Erwartung. Wichtig sind dabei Randfälle: die 0, negative Zahlen, gleiche Eingaben. Und weil Funktionen einzeln aufrufbar sind, lassen sie sich auch einzeln testen — noch ein Grund, Programme aus Funktionen zu bauen. Große Softwareprojekte automatisieren das sogar: Tausende Testfälle laufen dort nach jeder Änderung von selbst.
